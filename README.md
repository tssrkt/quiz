# Quiz / Викторины

Базовый каркас русскоязычного статического сайта викторин. Проект работает без сборщика, Node.js и собственного бэкенда: используются только HTML, CSS, JavaScript и JSON. Он рассчитан на публикацию в GitHub Pages по адресу `https://tssrkt.github.io/quiz/` и редактирование контента через Pages CMS.

## Структура

```text
index.html                     главная страница
quizzes.html                   каталог викторин
quiz.html                      заглушка будущего прохождения викторины
contacts.html                  информация о проекте и контакты
css/style.css                  общие стили и адаптивность
js/common.js                   меню, текущий год, копирование ЮMoney
js/quizzes.js                  загрузка, проверка JSON и карточки
tests/catalog.test.js          тесты сортировки, группировки, URL и пагинации
data/quizzes/*.json            отдельные файлы викторин
data/quizzes/index.json        список файлов викторин для браузера
data/tags/*.json               отдельные файлы тегов
data/tags/index.json           список файлов тегов для браузера
img/covers/                    квадратные обложки
img/quiz/                      изображения вопросов
.pages.yml                     схема Pages CMS
```

Все URL внутри сайта относительные. Поэтому один и тот же код работает в корне локального сервера и в подпапке `/quiz/` на GitHub Pages.

## Локальный запуск

HTML-страницы можно открыть прямо из проводника, но браузеры запрещают `fetch` локальных JSON-файлов по протоколу `file://`. Для полноценной проверки каталога запустите в корне проекта любой простой HTTP-сервер, например встроенный в Python:

```powershell
python -m http.server 8000
```

Затем откройте `http://localhost:8000/`. Остановка сервера — `Ctrl+C`.

## Публикация через GitHub Pages

1. Откройте репозиторий на GitHub и перейдите в **Settings → Pages**.
2. В **Build and deployment** выберите **Deploy from a branch**.
3. Выберите ветку `main`, каталог `/(root)` и сохраните настройки.
4. После публикации сайт будет доступен по адресу `https://tssrkt.github.io/quiz/`.

Проект не требует workflow сборки: GitHub Pages публикует готовые статические файлы.

## Подключение Pages CMS

Конфигурация сверена с актуальной официальной документацией Pages CMS: коллекции используют формат JSON, ссылки между коллекциями — поле `reference`, а обложки и изображения вопросов — раздельные именованные media-источники.

1. Перейдите на [app.pagescms.org](https://app.pagescms.org/) и войдите через GitHub.
2. Установите GitHub App Pages CMS для аккаунта, которому принадлежит репозиторий.
3. Разрешите приложению доступ к репозиторию `tssrkt/quiz`.
4. Выберите репозиторий и ветку `main`. Pages CMS прочитает `.pages.yml` из корня.
5. В панели появятся универсальные коллекции **Теги** и **Викторины**.

Pages CMS сохраняет правки непосредственно в выбранную ветку репозитория отдельными коммитами. Перед использованием панели убедитесь, что такой способ публикации подходит вашему процессу работы.

## Добавление контента

### Новый тег

В Pages CMS откройте **Теги**, создайте запись и заполните название, уникальный `slug`, порядок и видимость. `slug` состоит из строчных латинских букв, цифр и дефисов. После сохранения добавьте имя созданного файла (например, `history.json`) в `data/tags/index.json`.

### Новая викторина

Откройте **Викторины**, создайте запись, заполните метаданные, выберите один или несколько тегов и добавьте вопросы. Для каждого вопроса:

- задайте уникальный внутри викторины `id`;
- добавьте от 2 до 6 непустых вариантов;
- скопируйте один вариант без изменений в поле `correct_answer`;
- обязательно заполните объяснение;
- для содержательного изображения заполните `image_alt`;
- при необходимости укажите источник, автора и полную ссылку.

После сохранения добавьте имя нового файла (например, `history-basics.json`) в `data/quizzes/index.json`. Статический браузер не умеет перечислять содержимое каталога, поэтому эти два индексных файла служат явными реестрами. Pages CMS редактирует сами коллекции универсально, однако обновление реестра на первом этапе выполняется вручную.

### Изображения

Поле обложки загружает файлы в `img/covers`, поле изображения вопроса — в `img/quiz`. Для обложек рекомендуется квадратное изображение. CMS сохраняет относительные публичные пути, совместимые с GitHub Pages в подпапке проекта.

## Формат JSON

Тег:

```json
{"name":"Животные","slug":"animals","order":20,"published":true}
```

Викторина содержит поля `slug`, `title`, `published`, `publication_date`, `short_description`, `intro`, `cover`, `tags` и `questions`. Число вопросов нигде отдельно не хранится: карточка вычисляет его как `questions.length`.

Каждый объект в `questions` содержит `id`, `question`, необязательные `image` и `image_alt`, массив `answers`, строку `correct_answer`, обязательное `explanation` и необязательные `image_source`, `image_author`, `image_source_url`. Полный пример находится в `data/quizzes/horse-colors.json`.

Клиентский валидатор проверяет обязательные поля, уникальность `id` вопросов, 2–6 ответов, точное наличие правильного ответа в массиве, объяснение и корректный массив тегов. Повреждённый файл пропускается, а причина выводится в консоль разработчика, не нарушая работу остальных карточек.

## Каталог

Каталог показывает только опубликованные викторины и опубликованные теги. Тег «Все» создаётся программно. Счётчики вычисляются по загруженным опубликованным викторинам.

Выбранный тег не скрывает остальные записи: сначала выводятся викторины с этим тегом, затем остальные. Обе группы независимо сортируются по новым датам, старым датам или русскому алфавиту, после чего объединённый список разбивается на страницы по 25 карточек.

Состояние хранится в параметрах `tag`, `sort` (`new`, `old`, `az`) и `page`. Посторонние URL-параметры сохраняются. Кнопки браузера «Назад» и «Вперёд» восстанавливают состояние каталога.

Проверки чистой логики запускаются без установки зависимостей:

```powershell
node tests/catalog.test.js
```

## Ограничения первого этапа

- Нет движка прохождения викторины и отдельной страницы результата.
- После создания JSON через CMS имя файла нужно вручную добавить в соответствующий `index.json`.
- Уникальность `slug` между файлами и совпадение `correct_answer` CMS не может проверить динамически; эти условия контролируются автором и клиентским валидатором.
- Прямое открытие по `file://` показывает страницы, но каталог JSON требует локального HTTP-сервера из-за политики безопасности браузера.
- Страница `quiz.html` пока является заглушкой: движок прохождения викторины не реализован.
